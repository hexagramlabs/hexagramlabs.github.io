Locus Obscurus

2025/10/19

From the simple pinhole camera to our own eyes, it is a reoccurring theme to model three dimensional space and compress information. In some ways, this is the essence of both vision and computing. To achieve that end, however, an eye or lens must reflect whatever it sees from the physical realm, into a two-dimensional space.

This is also true with a “lens-less,” pinhole camera, which projects a map of coordinates from the 3-dimensional world, into 2-dimensional form. Understanding projective geometry is useful here.

Every frame observed in reality is a function that outputs the information necessary to describe or map that frame. If I take a photograph of a desk, I have made record the dimensions of that desk, the colors, textures, boundaries, shadows, vanishing point, and so on. Much of that information can be translated into an image. We can model this mathematically.

The focal length, principal point, skew parameter, and pinhole size can all be thought of as intrinsic factors. They are unchanging factors that are reusable from one scene to another.

However, there is also an extrinsic matrix–the parameters that map 3-dimensional world-space to 2-dimensional world-space. This parameter set changes when the camera position changes. The R and t parameters represent the homogenous matrix transformation and change from the world coordinate system to the camera’s coordinate system.

The matrix performs a rotation translation, the matrix product of a projective and homogenous transformation.
